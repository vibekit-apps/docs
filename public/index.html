VibeKit Documentation

Build and deploy full-stack apps with AI. One API call, live URL in minutes. Apps are hosted as Docker containers at {name}.vibekit.bot.

1 Quickstart

Sign in with GitHub

The fastest way to get started. Sign in via the web to get your API key and access the Dashboard.

https://vibekit.bot/auth/github

Or register via API:

curl -X POST https://vibekit.bot/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email": "you@example.com"}'

Submit a Task

curl -X POST https://vibekit.bot/api/v1/task \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Build a todo app with dark mode"}'

Check Status & Get URL

curl https://vibekit.bot/api/v1/task/TASK_ID \
  -H "Authorization: Bearer YOUR_API_KEY"

Your app will be live at {name}.vibekit.bot

Authentication

VibeKit supports two authentication methods:

1. GitHub Sign-In (Web)

Sign in with your GitHub account at /auth/github. This gives you access to the Dashboard and your API key.

2. API Key (Bearer Token)

All API requests require a Bearer token in the Authorization header:

Authorization: Bearer vk_your_api_key

Register via API

POST /api/v1/auth/register 
{"email": "you@example.com"}

Returns your API key. Free plan includes 3 sessions/month.

Tasks

Submit Task

POST /api/v1/task 
{
  "prompt": "Build a landing page",
  "skills": ["solana"],        // optional
  "projectId": "abc123",      // optional: iterate
  "callbackUrl": "https://..." // optional: webhook
}

Each task consumes one session.

Get Task Status

GET /api/v1/task/:taskId 
{
  "taskId": "task_abc123",
  "status": "complete",  // pending | running | complete | error
  "deployUrl": "https://myapp.vibekit.bot",
  "projectId": "abc123"
}

Hosting API

Apps are deployed as Docker containers and served at {name}.vibekit.bot. Manage deployments programmatically.

Deploy

POST /api/v1/hosting/deploy

Deploy a new app or redeploy an existing one.

curl -X POST https://vibekit.bot/api/v1/hosting/deploy \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "my-app", "repo": "user/repo"}'

List Apps

GET /api/v1/hosting/apps

Returns all apps for the authenticated user.

App Detail

GET /api/v1/hosting/app/:id

Returns app details including infrastructure topology (container status, resource usage, domain config).

{
  "id": "app_abc123",
  "name": "my-app",
  "url": "https://my-app.vibekit.bot",
  "status": "running",
  "infra": { /* topology details */ }
}

App Actions

POST /api/v1/hosting/app/:id/:action

Available actions:

restart stop start redeploy delete 
curl -X POST https://vibekit.bot/api/v1/hosting/app/APP_ID/restart \
  -H "Authorization: Bearer YOUR_API_KEY"

Resize App

POST /api/v1/hosting/app/:id/resize

Scale container resources for an app.

curl -X POST https://vibekit.bot/api/v1/hosting/app/APP_ID/resize \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"cpu": 2, "memory": "1G"}'

AI Agent

The AI Agent handles complex multi-step tasks — building, debugging, and deploying autonomously.

Start Agent Task

POST /agent 
curl -X POST https://vibekit.bot/agent \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Build and deploy a blog with auth"}'

Agent Activity

GET /agent/activity

Returns currently active agent tasks and their live status.

Agent History

GET /agent/history

Returns completed agent task history.

Agent Control

POST /agent/control

Send control commands to a running agent (pause, resume, cancel).

curl -X POST https://vibekit.bot/agent/control \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"taskId": "task_abc", "action": "cancel"}'

Skills

Skills inject domain-specific knowledge into code generation. Community-contributed via skills-registry.

GET /api/v1/skills
solana Wallet connection, dApp development, Anchor programs ✓ verified

Want to add a skill? Submit a PR to the registry.

Webhooks

Get notified when tasks complete. Include callbackUrl in your task submission.

{
  "taskId": "task_abc123",
  "status": "complete",
  "deployUrl": "https://myapp.vibekit.bot"
}

Signature Verification: Webhooks include X-VibeKit-Signature header. Verify with HMAC-SHA256 using your API key.

CLI

# Install
npm install -g vibekit-cli

# Login
vibekit login

# Build something
vibekit build "Create a blog with markdown"

# Check status
vibekit status task_abc123

MCP Server

Use VibeKit as an MCP tool with Claude Desktop.

npm install -g vibekit-mcp

Add to Claude Desktop config:

{
  "mcpServers": {
    "vibekit": {
      "command": "vibekit-mcp",
      "env": {"VIBEKIT_API_KEY": "vk_..."}
    }
  }
}

Advanced Workflows

For complex features, combine VibeKit with Antfarm multi-agent workflows.

Antfarm provides plan → implement → verify → test → review cycles with specialized AI agents. VibeKit handles deployment.

# Antfarm develops the feature
antfarm workflow run feature-dev "Add Stripe checkout"

# VibeKit deploys when ready
curl -X POST https://vibekit.bot/api/v1/task \
  -H "Authorization: Bearer $KEY" \
  -d '{"prompt": "Deploy latest", "repo": "user/app"}'

Use VibeKit for quick iterations, Antfarm for features needing verification gates.
